# Data Dictionary

| Column | Meaning |
|---|---|
| system_name | Name of the infrastructure, institution, supply chain, digital platform, utility, public service, or planning unit |
| sector | Sector such as energy, water, health, transport, digital systems, finance, food systems, public services, or governance |
| system_type | System category such as critical infrastructure, high-speed digital platform, hospital system, utility network, lean supply chain, or emergency-response system |
| coupling_strength_index | Degree of dependence between system parts |
| time_compression_index | How little time exists between disturbance and consequence |
| sequence_rigidity_index | Degree to which processes must happen in fixed order |
| limited_substitution_index | Lack of substitute components, routes, suppliers, staff, procedures, or systems |
| interactive_complexity_index | Unexpected interactions among components, technologies, institutions, and human operators |
| hidden_dependency_index | Dependencies that are poorly mapped or become visible only during failure |
| critical_node_importance_index | Degree to which affected nodes support many other essential functions |
| buffering_index | Spare capacity, inventory, backup power, storage, emergency funds, staff margin, or time buffer |
| modularity_index | Ability to isolate failure and prevent system-wide spread |
| redundancy_index | Credible alternate pathways for preserving essential function |
| adaptive_authority_index | Ability of local or responsible actors to pause, reroute, improvise, and respond |
| fallback_capacity_index | Manual procedures, backup systems, substitute services, recovery capacity, and restoration capability |
