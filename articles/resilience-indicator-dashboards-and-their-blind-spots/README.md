# Resilience Indicator Dashboards and Their Blind Spots

This article directory supports the Risk & Resilience article:

**Resilience Indicator Dashboards and Their Blind Spots**

The workflows model how indicator coverage, dashboard usability, data quality, update frequency, disaggregation strength, equity visibility, uncertainty transparency, action linkage, community validation, governance accountability, threshold clarity, stress-performance integration, false precision, proxy dependence, missing data, aggregation loss, scale-boundary error, dashboard theater, and blind-spot severity shape resilience-dashboard reliability.

## Analytical Focus

The article examines:

- Resilience indicator dashboards
- Dashboard blind spots
- False precision
- Proxy dependence
- Composite scores
- Aggregation loss
- Missing data and data-quality gaps
- Scale and boundary problems
- Threshold uncertainty
- Equity visibility
- Distributional inequality
- Community validation
- Dashboard governance
- Action linkage
- Dashboard theater
- Performative accountability
- Uncertainty transparency
- Stress-performance integration

## Suggested Input Dataset

Expected file:

```text
resilience_dashboard_blind_spots_panel.csv
```

Each `*_index` field should be normalized from 0 to 1.

Higher values should mean more of the named property. For example:

- `indicator_coverage_index`: higher = stronger coverage of relevant resilience domains
- `uncertainty_transparency_index`: higher = clearer communication of uncertainty
- `false_precision_risk_index`: higher = greater risk of overconfident dashboard interpretation
- `dashboard_theater_risk_index`: higher = greater risk that reporting replaces action
