# Data Dictionary

| Column | Meaning |
|---|---|
| system_name | Name of the system being assessed |
| sector | Sector such as energy, water, transport, healthcare, digital, food, logistics, or public services |
| system_type | Type of system, such as infrastructure, cyber-physical, supply chain, public institution, or ecological buffer |
| normal_performance_index | Routine performance under ordinary conditions |
| cost_efficiency_index | Cost performance or resource efficiency |
| utilization_pressure_index | Pressure from high utilization or limited spare capacity |
| slack_capacity_index | Available slack, surge capacity, or operating margin |
| redundancy_capacity_index | Backup systems, alternate routes, substitute suppliers, or duplicate capacity |
| flexibility_capacity_index | Ability to adapt or reconfigure under stress |
| modularity_index | Degree to which failures can be isolated |
| dependency_visibility_index | Visibility into direct and indirect dependencies |
| dependency_concentration_index | Exposure to single points of failure or concentrated dependencies |
| tight_coupling_index | Degree of timing dependence and rapid failure propagation |
| maintenance_vulnerability_index | Risk from deferred maintenance or weak repair capacity |
| cyber_exposure_index | Exposure to digital, software, platform, or cyber-physical disruption |
| climate_hazard_exposure_index | Exposure to climate hazards or nonstationary environmental conditions |
| service_criticality_index | Importance of the system for essential service continuity |
| governance_capacity_index | Strength of governance, coordination, accountability, and planning |
| recovery_capacity_index | Ability to restore service after disruption |
| equity_vulnerability_index | Degree to which failure would disproportionately affect vulnerable communities |
