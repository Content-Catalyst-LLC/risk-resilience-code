# Designing for Resilience Rather Than Optimization Alone

This article directory supports the Risk & Resilience article:

**Designing for Resilience Rather Than Optimization Alone**

The workflows model the difference between systems optimized narrowly for normal-period efficiency and systems designed to preserve function under disturbance.

## Analytical Focus

The article examines:

- Optimization-induced fragility
- High utilization pressure
- Low slack capacity
- Redundancy and backup capacity
- Flexibility and modularity
- Dependency concentration
- Tight coupling
- Maintenance vulnerability
- Cyber exposure
- Climate hazard exposure
- Service criticality
- Governance capacity
- Recovery capacity
- Equity vulnerability

## Suggested Input Dataset

Expected file:

```text
resilience_design_system_panel.csv
```

Each `*_index` field should be normalized from 0 to 1.
