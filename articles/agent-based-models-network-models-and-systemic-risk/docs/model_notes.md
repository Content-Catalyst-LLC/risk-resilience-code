# Model Notes

This article uses conceptual agent-based and network systemic-risk models.

The scores are not objective predictions of systemic failure. They are diagnostic tools for making interdependence, contagion pathways, agent behavior, and resilience buffers visible.

## Core Concepts

- **Agent heterogeneity**: agents differ in exposure, vulnerability, capacity, governance support, redundancy, and behavioral sensitivity.
- **Network topology**: systemic risk depends on connection structure, dependency weights, centrality, modularity, and hidden shared dependencies.
- **Behavioral amplification**: agents may amplify stress through panic, hoarding, fire sales, withdrawal, imitation, overload, or defensive behavior.
- **Contagion and cascades**: local stress can spread through network links and trigger wider system failure.
- **Resilience buffers**: capacity, governance, redundancy, modularity, and recovery capability can dampen systemic propagation.
- **Systemic fragility**: risk produced by the interaction of vulnerable nodes, central positions, strong dependencies, limited buffers, and destabilizing behavior.

## Interpretation

A system can appear resilient at the component level while remaining fragile at the network level.

The model is most useful for scenario analysis, network stress testing, systemic-risk diagnosis, infrastructure dependency review, financial contagion analysis, supply-chain fragility mapping, and model-governance discussion.
