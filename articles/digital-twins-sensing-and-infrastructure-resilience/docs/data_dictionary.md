# Data Dictionary

| Column | Meaning |
|---|---|
| system_name | Name of the digital twin or infrastructure system being assessed |
| sector | Sector such as energy, water, transport, healthcare, communications, logistics, or public services |
| infrastructure_type | Type of infrastructure represented by the twin |
| sensing_coverage_index | Coverage of relevant assets, hazards, service conditions, and operational states |
| data_quality_index | Accuracy, completeness, calibration, and reliability of data |
| data_timeliness_index | Timeliness of data capture, transfer, and updating |
| model_validation_index | Strength of model testing, validation, and fitness for intended use |
| decision_integration_index | Degree to which twin outputs are connected to real decisions |
| dependency_visibility_index | Visibility into direct and indirect infrastructure dependencies |
| predictive_maintenance_usefulness_index | Usefulness for maintenance prioritization and lifecycle asset management |
| climate_adaptation_usefulness_index | Usefulness for climate risk analysis and adaptation planning |
| service_continuity_relevance_index | Relevance to preserving essential services under stress |
| cyber_risk_index | Exposure to cyber compromise, data corruption, access risk, or operational disruption |
| platform_dependency_index | Exposure to proprietary lock-in, vendor dependency, or fragile platform reliance |
| model_uncertainty_index | Uncertainty, incompleteness, or possible misuse of the model |
| equity_coverage_index | Degree to which vulnerable communities and unequal service exposure are represented |
| governance_capacity_index | Institutional ability to maintain, audit, secure, and act on the twin |
| public_accountability_index | Transparency, oversight, public-interest safeguards, and accountability |
