# Data Dictionary

| Column | Meaning |
|---|---|
| system_name | Name of the AI, automation, decision-support, or algorithmic governance system |
| jurisdiction | Geographic or institutional jurisdiction |
| system_domain | System domain such as public service, infrastructure, health, finance, cyber, emergency management, supply chain, or environmental monitoring |
| model_capability_index | Useful model performance for its intended resilience purpose |
| institutional_governance_index | Strength of institutional policies, roles, approval, review, and accountability |
| data_quality_index | Accuracy, completeness, timeliness, representativeness, and relevance of input data |
| human_oversight_index | Meaningful human review, override, authority, training, and decision rights |
| auditability_index | Traceability of data, model versions, logs, parameters, outputs, and decisions |
| system_robustness_index | Resilience under stress, outage, adversarial pressure, and unusual conditions |
| equity_testing_index | Testing for group-level error, exclusion, proxy discrimination, accessibility, and distributional harms |
| security_control_index | Cybersecurity, access control, adversarial testing, API security, and model asset protection |
| fallback_capacity_index | Manual fallback, degraded-mode operation, redundancy, and recovery procedures |
| public_contestability_index | Ability of affected people to understand, challenge, appeal, and correct decisions |
| monitoring_maturity_index | Post-deployment monitoring for performance, drift, incidents, disparities, and downstream harms |
| incident_response_index | Ability to detect, report, investigate, correct, and learn from algorithmic incidents |
| vendor_accountability_index | Contractual and practical ability to audit, test, exit, and hold vendors accountable |
| opacity_risk_index | Risk that the system is hidden, unexplained, proprietary, or institutionally unaccountable |
| bias_severity_index | Severity of unequal error, exclusion, proxy discrimination, or group-level harm |
| model_drift_risk_index | Risk that performance degrades as data, behavior, policy, or environments change |
| automation_dependency_index | Degree of institutional reliance on automation without adequate fallback capacity |
| vendor_concentration_risk_index | Risk from dependence on a small number of vendors, platforms, cloud systems, or model providers |
| cyber_exposure_index | Cyber risk from model infrastructure, data pipelines, APIs, integrations, prompts, or tool access |
| legitimacy_risk_index | Risk that affected people lose trust because the system is unfair, opaque, inaccessible, or unaccountable |
